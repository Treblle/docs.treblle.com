Treblle auto-generates API documentation in real-time, ensuring accuracy and up-to-date information for every release. 

It includes details such as available endpoints, authentication methods, request and response formats, and other essential technical specifications required to successfully integrate and use the API.

## How to View API Docs

1. Click on **API Docs** in the left navigation bar.

   <img
     src="/platform/api-documentation/api-documentation.png"
     width="200"
     style={{ borderRadius: "8px" }}
   />

2. The API Docs Overview will display the following:

    <img
      src="/platform/api-documentation/docs-dashboard.png"
      width="800"
      style={{ borderRadius: "8px" }}
   />

 - **Load time**: Time it takes to load the API data.
 - **Response size**: Size of the API responses.
 - **Base URL**: The base URL for your API.
 - **[API score](https://docs.treblle.com/treblle/api-governance/)**: A score that represents the health and quality of your API.

    <img
      src="/platform/api-documentation/api-score.png"
      width="400"
      style={{ borderRadius: "8px" }}
   />

 - **Format**: The format of the API (e.g., JSON).
 - **Total number of endpoints**: Total count of API endpoints.
 - **API settings**: Includes API details, invite members, API ID, and SDK token.
 - **[API catalog](https://docs.treblle.com/features/api-catalog/)**: A repository to publish and manage API documentation for collaboration without granting full API access.

    <img
      src="/platform/api-documentation/api-catalog.png"
      width="400"
      style={{ borderRadius: "8px" }}
    />

## Authentication

Treblle uses standardized authentication to integrate with client libraries. 

To authenticate your requests, provide the following details:

- **Type:** Specify the authentication method (e.g., Bearer Token, API Key).
- **Value:** Provide the authentication value.
- **Location:** Indicate where the authentication should be applied (e.g., Header, Query Parameter).

   <img
     src="/platform/api-documentation/authentication.png"
     width="800"
     style={{ borderRadius: "8px" }}
   />

## OpenAPI Specification

Treblle auto-generates API documentation with OpenAPI Specification (OAS) support for every project. Supported formats include JSON and YAML.

To open the documentation in Swagger, click the **Open in Swagger** button on the OpenAPI Specification page.

<img
  src="/platform/api-documentation/openapi.png"
  width="800"
  style={{ borderRadius: "8px" }}
/>


:::note
A new OpenAPI Specification URL is generated each time you visit the documentation page. This URL is valid for one hour and expires afterward.
:::

## Alfred AI 

**Alfred** is Treblle's assistant that helps with your API workflows by using your API documentation to provide real-time answers.

:::note
Alfred only accesses your API documentation and does not store or analyze your data. It helps improve your documentation and integration process based on your current API setup.
:::
